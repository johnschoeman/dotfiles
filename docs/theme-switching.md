# Theme Switching

Runtime theme switching across Alacritty, Zellij, Helix, Waybar, Rofi, Yazi, Mako, and Hyprlock via a single script, bound to `Super+T`.

## Supported Themes

| Name | Notes |
|---|---|
| `catppuccin_frappe` | Default |
| `catppuccin_mocha` | |
| `gruvbox_dark` | |

| `solarized_light` | Only light theme |
| `tokyo_night` | |

The canonical name format uses underscores (Alacritty convention). Other tools need different formats ŌĆö the script handles translation.

## Architecture

```
Super+T ŌåÆ scripts/theme-select.sh#launch_rofi ŌåÆ rofi picker ŌåÆ scripts/theme-select.sh#set_theme
```

### The NixOS Problem

NixOS generates config files into `/nix/store/` (read-only). Runtime theme switching requires mutable configs. The solution: `mkOutOfStoreSymlink` ŌĆö home-manager creates a symlink from `~/.config/<app>/config` to a file in the dotfiles repo instead of the nix store. The script then modifies the repo file directly.

This pattern is used for Zellij (`zellij/config.kdl`), Helix (`helix/config.toml`, `helix/languages.toml`), and Yazi (`yazi/theme.toml`). Alacritty uses a different approach ŌĆö it imports a `current-theme.toml` file that gets overwritten.

The `catppuccin/nix` module was previously used for automatic theming of some programs, but it conflicted with runtime switching ŌĆö rebuilds would reset theme changes and some programs were nix-managed while others were script-managed. It was removed so that `theme-select` is the single source of truth.

### Gitignored Theme Files

Four mutable theme files are gitignored to prevent theme switches from polluting `git status`:
- `waybar/colors.css` ŌĆö relative symlink to `themes/<name>.css`
- `rofi/colors.rasi` ŌĆö relative symlink to `themes/<name>.rasi`
- `hyprlock/colors.conf` ŌĆö relative symlink to `themes/<name>.conf`
- `mako/config` ŌĆö generated by concatenating `base.conf` + `themes/<name>.conf`

On a fresh build (or if any of these files are missing), the `themeDefaults` activation script in `home.nix` creates them using the current theme (read from `~/.config/alacritty/current-theme-name`, defaulting to `catppuccin_frappe`). This runs after `linkGeneration` so all `mkOutOfStoreSymlink` links are in place first.

### Per-Tool Mechanics

**Alacritty** ŌĆö `alacritty.toml` imports `current-theme.toml` via `general.import`. The script copies the selected theme file over `current-theme.toml` and touches `alacritty.toml` to trigger inotify-based live reload. Theme files come from `pkgs.alacritty-theme`, symlinked to `~/.config/alacritty/themes/`. A home-manager activation script seeds the default on first run.

**Zellij** ŌĆö `sed` replaces the `theme "..."` line in `config.kdl`. Uses hyphen-separated names (`catppuccin-frappe`). No live reload; applies to new sessions only.

**Helix** ŌĆö `sed` replaces the `theme = "..."` line in `config.toml`. Uses Helix built-in theme names, which mostly match the canonical underscore format. Two exceptions need mapping: `gruvbox_dark` ŌåÆ `gruvbox`, `tokyo_night` ŌåÆ `tokyonight`. No automatic live reload, but `space t` (`:config-reload`) picks up changes in a running instance.

**Waybar** ŌĆö `style.css` imports `colors.css` via `@import`. The script creates a relative symlink from `colors.css` ŌåÆ `themes/<name>.css`, then sends `SIGUSR2` to Waybar for instant reload. Theme CSS files map each theme's palette onto Catppuccin semantic color names (`@define-color base`, `@define-color text`, etc.), so `style.css` uses one set of variable names regardless of theme. `colors.css` is gitignored.

**Yazi** ŌĆö reads `theme.toml` from `~/.config/yazi/`, which is a mutable symlink (`mkOutOfStoreSymlink`) to `yazi/theme.toml` in the repo. Yazi watches for config changes and reloads instantly.

**Rofi** ŌĆö `theme.rasi` imports `colors.rasi` via `@import`. The script creates a relative symlink from `colors.rasi` ŌåÆ `themes/<name>.rasi`. Same color variable pattern as Waybar ŌĆö each theme maps its palette onto Catppuccin semantic names. Config references the repo file directly (string path in `programs.rofi.theme`), so it stays mutable. Applies on next rofi launch. `colors.rasi` is gitignored.

**Mako** ŌĆö Mako's config format doesn't support imports, so the script concatenates a shared `mako/base.conf` (layout/timing settings) with a per-theme color file from `mako/themes/`, writing the result to `mako/config`. The config file is a mutable symlink (`mkOutOfStoreSymlink`). `makoctl reload` applies changes instantly. Theme files contain global colors plus an `[urgency=critical]` section for the critical border color. `mako/config` is gitignored (generated file).

**Hyprlock** ŌĆö `hyprlock.conf` sources `colors.conf` via hyprlang's `source` directive. The script creates a relative symlink from `colors.conf` ŌåÆ `themes/<name>.conf`. Three variables: `$font_color` (text), `$inner_color` (overlay1), `$outer_color` (crust). The config is a mutable symlink (`mkOutOfStoreSymlink`). Hyprlock reads config fresh each launch, so changes apply on the next lock ŌĆö no reload needed. `colors.conf` is gitignored.

## Name Translation

Each tool has its own naming convention. The script starts with the canonical underscore name and translates:

| Tool | Format | Example | How |
|---|---|---|---|
| Alacritty | `snake_case` | `catppuccin_frappe` | Canonical, no translation |
| Zellij | `kebab-case` | `catppuccin-frappe` | `${name//_/-}` |
| Helix | Varies | `tokyonight` | Associative array for exceptions, passthrough otherwise |
| Waybar | `kebab-case` | `catppuccin-frappe` | Reuses Zellij's translated name for file lookup |
| Rofi | `kebab-case` | `catppuccin-frappe` | Reuses Zellij's translated name for file lookup |
| Mako | `kebab-case` | `catppuccin-frappe` | Reuses Zellij's translated name for file lookup |
| Hyprlock | `kebab-case` | `catppuccin-frappe` | Reuses Zellij's translated name for file lookup |

## File Layout

```
scripts/theme-select.sh           # Main script
helix/config.toml                 # Mutable helix config (mkOutOfStoreSymlink)
helix/languages.toml              # Mutable helix languages (mkOutOfStoreSymlink)
zellij/config.kdl                 # Mutable zellij config (mkOutOfStoreSymlink)
yazi/theme.toml                   # Mutable yazi theme (mkOutOfStoreSymlink)
waybar/colors.css                 # Symlink to active theme (gitignored)
waybar/themes/*.css               # Per-theme color definitions
waybar/style.css                  # Imports colors.css
rofi/theme.rasi                   # Layout, imports colors.rasi
rofi/colors.rasi                  # Symlink to active theme (gitignored)
rofi/themes/*.rasi                # Per-theme color definitions
mako/config                       # Generated: base + colors (gitignored)
mako/base.conf                    # Non-color settings (layout, timeouts)
mako/themes/*.conf                # Per-theme color definitions
hyprlock/hyprlock.conf            # Main config, sources colors.conf
hyprlock/colors.conf              # Symlink to active theme (gitignored)
hyprlock/themes/*.conf            # Per-theme color definitions
nixos/home/alacritty.nix          # Alacritty config + activation script
nixos/home/helix.nix              # Helix config (mkOutOfStoreSymlink)
nixos/home/rofi.nix               # Rofi config (string path to repo theme)
nixos/home/yazi.nix               # Yazi config (mkOutOfStoreSymlink)
nixos/home/hyprland.nix           # Mako config (mkOutOfStoreSymlink)
nixos/home/zellij.nix             # Zellij config (mkOutOfStoreSymlink)
```

## Adding a New Theme

1. Verify the theme exists in `~/.config/alacritty/themes/` (from `pkgs.alacritty-theme`)
2. Create `waybar/themes/<kebab-name>.css` mapping the theme's palette to Catppuccin semantic color names
3. Create `rofi/themes/<kebab-name>.rasi` mapping the theme's palette to Catppuccin semantic color names
4. Create `mako/themes/<kebab-name>.conf` with background-color, text-color, border-color, and `[urgency=critical]` border-color + default-timeout
5. Create `hyprlock/themes/<kebab-name>.conf` with `$font_color` (text), `$inner_color` (overlay1), `$outer_color` (crust)
6. Verify the theme exists as a Zellij built-in and a Helix built-in
7. If the Helix built-in name differs from the underscore name, add an entry to `helix_map` in the script
8. Add the name to the `CURATED` array in `scripts/theme-select.sh`
