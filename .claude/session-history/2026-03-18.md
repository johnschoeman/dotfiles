# 2026-03-18

## Set cursor theme to Bibata Modern Classic

**Thread:** Set an explicit cursor theme — the system was using whatever default was available. Bibata Modern Classic is a material-design cursor with black fill and thin outline.

Added `bibata-cursors` package and a `home.pointerCursor` block in `hyprland.nix` — sets theme name, package, size 12, and enables both GTK and Hyprcursor integration. Removed the manual `XCURSOR_SIZE` and `HYPRCURSOR_SIZE` env vars from `hyprland.conf` since `home.pointerCursor` sets those automatically.

Hit the recurring zellij `config.kdl.backup` collision again during rebuild — third time this has happened. Fixed it permanently by adding `force = true` to the zellij config file declaration in `zellij.nix`, which tells home-manager to overwrite in place instead of backing up.

Cursor theme required a full Hyprland logout/login to take effect (env vars set at session start).

## Reorganize and consolidate skills

**Thread:** Clean up skill naming for consistency and consolidate redundant skills.

Renamed six skills for better grouping and shorter invocations: `close-day` → `wrap-day`, `project-init` → `setup-claude`, `planning-check` → `planning`, `capture-content` → `content-capture`, `synthesize-learning` → `content-synthesize`. The `content-*` pair now sorts together alphabetically.

Consolidated `make-commit` and `suggest-commit` into a single `/commit` skill. Default behavior is suggest-only (writes `commit-msg.txt`, shows files to stage, reminds user). A `--run` flag stages and commits — but checks CLAUDE.md first and falls back to suggest mode if the repo says the user manages commits. This solves the cross-repo problem: repos with strict git rules are protected, repos without them get the convenience of auto-commit when desired.

Also rewrote `setup-claude` (formerly `project-init`) with two modes: fresh setup from scratch, or audit mode that analyzes existing infrastructure and suggests improvements to align with the standard layout. And created a new `wrap-session` skill that chains `/update-session-log` → `/update-knowledge` (conditional) → `/commit` as an end-of-session workflow.

Updated all cross-references: `wrap-session` now points to `/commit`, `setup-claude` references the renamed skills (`/planning`, `/content-capture`, `/content-synthesize`), and internal error messages use the new names.

**Open:** All skill changes uncommitted (14 deleted files, 7 new directories). Previous cursor theme and zellij force-fix changes also uncommitted.
