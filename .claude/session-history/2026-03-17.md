# 2026-03-17

## Add mako to theme-select

**Thread:** Integrate mako (notification daemon) into the runtime theme switching system — the last program with hardcoded Catppuccin Frappe colors.

Mako's config format doesn't support imports like waybar/rofi, so used a concatenation approach: a shared `mako/base.conf` (layout/timing settings) gets concatenated with a per-theme color file from `mako/themes/` to produce the active `mako/config`. The script runs `makoctl reload` for instant application.

Key design decision during implementation: the plan had theme files concatenated before the base file (`cat theme base`), but this breaks mako's INI-like section parsing — global settings from base would end up inside the theme's `[urgency=critical]` section. Reversed to `cat base theme` and moved the critical timeout (`default-timeout=10000`) into each theme file so there's only one clean `[urgency=critical]` section at the end of the final config.

In `hyprland.nix`, replaced the full `services.mako.settings` block with just `services.mako.enable = true` plus an `mkOutOfStoreSymlink` for the config — same pattern used for zellij, helix, and yazi. Build succeeded without conflicts between the service and the symlinked config.

**Open:** Hyprlock is the only remaining program with hardcoded theme colors. All theme-select targets now covered: Alacritty, Zellij, Helix, Waybar, Rofi, Yazi, Mako.
