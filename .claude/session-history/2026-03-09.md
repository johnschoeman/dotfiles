## Create global project-init skill

**Thread:** Consolidate the steps for setting up a new project for Claude Code into a single `/project-init` skill — analyze codebase, generate CLAUDE.md, create `.claude/` infrastructure, and optionally scaffold context files for planning/content capture skills.

Planned the skill in plan mode first, then implemented as a single `SKILL.md` at `claude/skills/project-init/`. The skill does its own codebase analysis rather than wrapping `/init` — cleaner than generating then post-processing. Key design choices: the Git Workflow section is verbatim (identical text in every project, matching the dotfiles CLAUDE.md convention), KNOWLEDGE.md template matches what `/update-knowledge` expects, and `.claude/docs/` templates are offered as opt-in since not every repo needs planning or content capture.

Caught a nested code fence issue during review — the Git Workflow section contains a commit format code block inside a markdown code block. Fixed by using 5-backtick fences for the outer block.

**Open:** Skill is created but uncommitted. Not yet tested end-to-end in a real project.
