# 2026-03-16

## Nix formatter behavior and autoformatting decision

**Thread:** Investigated whether to use nixfmt or nixpkgs-fmt for formatting .nix files, and whether to keep autoformatting enabled.

Started by adding Thunderbird as an Outlook alternative for NixOS, which led to discovering that nixfmt was collapsing vertically-aligned inline comments to single-space format. Researched both nixfmt and nixpkgs-fmt to see if either could preserve vertical comment alignment.

Created test files and ran both formatters. Found that both nixfmt and nixpkgs-fmt produce identical output — they both collapse vertical spacing to a single space before `#` comments. This is documented in nixfmt's standard as explicitly out of scope: "no special care is taken to preserve vertical alignment in the AST or comments."

Decided to disable autoformatting entirely to preserve manual formatting preferences. Removed the "After editing or creating `.nix` files: Run `nixfmt` on them before finishing" instruction from CLAUDE.md. The `nixfmt-rfc-style` package remains installed for optional manual use.

Also cleaned up `.claude/settings.local.json` permissions — removed colons from Bash permissions (e.g., `Bash(lsof:*)` → `Bash(lsof *)`), removed duplicate git status permissions, reorganized for readability, and moved `nh os switch` to the deny list per CLAUDE.md guidance.

**Open:** Staged changes ready to commit. Decided against using Thunderbird after testing.
