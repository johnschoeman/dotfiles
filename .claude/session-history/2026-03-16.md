## Nix formatter behavior and autoformatting decision

**Thread:** Investigated whether to use nixfmt or nixpkgs-fmt for formatting .nix files, and whether to keep autoformatting enabled.

Started by adding Thunderbird as an Outlook alternative for NixOS, which led to discovering that nixfmt was collapsing vertically-aligned inline comments to single-space format. Researched both nixfmt and nixpkgs-fmt to see if either could preserve vertical comment alignment.

Created test files and ran both formatters. Found that both nixfmt and nixpkgs-fmt produce identical output — they both collapse vertical spacing to a single space before `#` comments. This is documented in nixfmt's standard as explicitly out of scope: "no special care is taken to preserve vertical alignment in the AST or comments."

Decided to disable autoformatting entirely to preserve manual formatting preferences. Removed the "After editing or creating `.nix` files: Run `nixfmt` on them before finishing" instruction from CLAUDE.md. The `nixfmt-rfc-style` package remains installed for optional manual use.

Also cleaned up `.claude/settings.local.json` permissions — removed colons from Bash permissions (e.g., `Bash(lsof:*)` → `Bash(lsof *)`), removed duplicate git status permissions, reorganized for readability, and moved `nh os switch` to the deny list per CLAUDE.md guidance.

**Open:** Staged changes ready to commit. Decided against using Thunderbird after testing.

## Extend theme-select to Waybar

**Thread:** Add Waybar as a third target for `theme-select`, so the rofi theme picker updates Alacritty, Zellij, and Waybar together.

Waybar imports a color file via CSS `@import` in `style.css`. The import path is baked into the nix store at build time, but waybar resolves the imported file at runtime — so the color file itself is already mutable. Created `waybar/colors.css` as the active color file (analogous to Alacritty's `current-theme.toml`) and `waybar/themes/` with CSS files for all 7 curated themes. Each theme file defines the same 26 `@define-color` variables. For non-Catppuccin themes, accent colors are mapped semantically from the source palette.

Updated `style.css` to import `colors.css` instead of `frappe.css`, and fixed a bug where `@subtext2` was referenced (doesn't exist — only `subtext0` and `subtext1`). In `theme-select`, added a block after the Zellij update that copies the matching theme file to `colors.css` and sends `SIGUSR2` to waybar for instant reload. If no matching waybar theme exists (uncurated theme), waybar keeps its current colors.

User tested with Rose Pine — waybar reloaded instantly with the new palette.

**Open:** Changes uncommitted. Needs one `nrs` to bake the new `@import` path, after which all future theme switches are instant. The old `frappe.css` is now unused but still in the repo.

## Remove catppuccin/nix, make theme-select the single source of truth

**Thread:** Eliminate the catppuccin/nix flake module so all theming flows through `theme-select` and mutable dotfiles.

The catppuccin/nix module (`catppuccin.enable = true; flavor = "frappe"`) created a hybrid where some programs were nix-managed (locked to frappe, reset on rebuild) and others were switched at runtime by `theme-select`. Removed the module entirely: stripped the flake input, both module imports, the global config block in home.nix, and the per-app opt-outs in alacritty.nix and helix.nix.

Two programs needed attention after removal. Yazi had its theme silently managed by the module — copied the 771-line catppuccin frappe `theme.toml` from the nix store into `yazi/theme.toml` and wired it as a `mkOutOfStoreSymlink` in yazi.nix. Yazi turned out to already support live theme switching through this mutable file. Rofi also lost its theming — the module was injecting a `@theme` directive. Created `rofi/theme.rasi` (layout) + `rofi/colors.rasi` (active colors), mirroring the waybar pattern. Added per-theme color files in `rofi/themes/` and wired rofi into `theme-select`. Used a string path in `programs.rofi.theme` so rofi reads the repo file directly (mutable) rather than a nix store copy.

Hit a home-manager activation error on first rebuild — a stale `config.kdl.backup` file was blocking the zellij symlink. Unrelated to the catppuccin changes, resolved by removing the backup file.

Updated KNOWLEDGE.md (theme approach decision + NixOS learnings) and docs/theme-switching.md (added rofi and yazi, updated architecture diagram, file layout, and "adding a new theme" checklist).

**Open:** Changes uncommitted. Mako is the last program with hardcoded frappe colors (in hyprland.nix) — candidate for future `theme-select` integration.
