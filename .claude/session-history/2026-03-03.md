## Switch session log to daily files

**Thread:** Move from a single `SESSION_LOG.md` to daily files in `session-history/YYYY-MM-DD.md` to keep files small and provide natural archiving.

The single-file approach had grown unwieldy — evolv-notes hit 34KB/663 lines in ~3 weeks. Daily files keep each file small, let the skill read only recent context, and provide natural time-based archiving.

Updated the `/update-session-log` skill to target `session-history/YYYY-MM-DD.md`, read recent history from the directory, and append multiple entries per day. Entry format stays the same but drops the date prefix (date is in the filename). Updated CLAUDE.md git workflow references in three repos (dotfiles, notes, evolv-notes) to use `/update-session-log` instead of referencing `SESSION_LOG.md` directly. Existing `SESSION_LOG.md` files left in place as valid history — no migration needed.

**Open:** Changes are uncommitted across three repos (dotfiles has the skill + CLAUDE.md; notes and evolv-notes each have a CLAUDE.md edit).

## Move git-analysis script to global

**Thread:** The `/git-analysis` skill had just been moved to a global skill in dotfiles, but still used a two-path approach — check for a local `analyze_colocations.sh`, otherwise fall back to inline bash commands. The notes repo had a sophisticated script with hardcoded notes-specific filtering; evolv-notes had nothing (used the inline fallback). Goal was to make one generic script that works for any repo.

Generalized the notes script into `dotfiles/scripts/analyze_colocations.sh` with `--strip-prefix PREFIX` and `--exclude PATTERN` flags so repos can customize without hardcoding. Default exclusions kept: `_archive/`, `.trash/`, `.obsidian/`. Dropped notes-specific daily note exclusions from defaults. Used a shell `preprocess` function for filtering before piping to awk — cleaner than trying to dynamically generate awk code with variable patterns.

Updated the skill to always call the script (no more inline fallback), and to check repo structure before running to decide whether `--strip-prefix` is needed. Deleted the notes-local copy and updated `notes/.claude/settings.local.json` permission to reference `~/dotfiles/scripts/analyze_colocations.sh`.

Tested in both dotfiles (generic) and notes (with prefix stripping + exclusions) — both produce correct output.

**Open:** Changes uncommitted in dotfiles (new script, updated skill) and notes (deleted script, updated settings).
