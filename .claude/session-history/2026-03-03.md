# 2026-03-03

## Switch session log to daily files

**Thread:** Move from a single `SESSION_LOG.md` to daily files in `session-history/YYYY-MM-DD.md` to keep files small and provide natural archiving.

The single-file approach had grown unwieldy — evolv-notes hit 34KB/663 lines in ~3 weeks. Daily files keep each file small, let the skill read only recent context, and provide natural time-based archiving.

Updated the `/update-session-log` skill to target `session-history/YYYY-MM-DD.md`, read recent history from the directory, and append multiple entries per day. Entry format stays the same but drops the date prefix (date is in the filename). Updated CLAUDE.md git workflow references in three repos (dotfiles, notes, evolv-notes) to use `/update-session-log` instead of referencing `SESSION_LOG.md` directly. Existing `SESSION_LOG.md` files left in place as valid history — no migration needed.

**Open:** Changes are uncommitted across three repos (dotfiles has the skill + CLAUDE.md; notes and evolv-notes each have a CLAUDE.md edit).
