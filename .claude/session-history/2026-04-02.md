## Configure npm + JFrog Artifactory auth with 1Password

**Thread:** Set up npm/pnpm authentication with JFrog Artifactory (`evolv.jfrog.io`) using 1Password CLI for token management, following the plan from a previous session.

Created `nixos/home/npm.nix` as a new home-manager module that writes `~/.npmrc` via `home.file`. Used `home.file` instead of `programs.npm.settings` because the scoped registry auth key (`//evolv.jfrog.io/.../:_authToken`) contains slashes and colons that don't work as Nix attribute names, and `programs.npm.enable` would try to install nodejs conflicting with the existing `nodejs_24` in home.packages. The `.npmrc` uses `${JFROG_NPM_TOKEN}` which npm interpolates at runtime from the environment variable.

Added a `jfrog-login` fish function that calls `op read` to load the token from 1Password into `JFROG_NPM_TOKEN`. The 1Password reference is `op://Personal/JFrog Artifactory Token/token` — needs to match actual vault/item/field names.

Rebuild succeeded on first try. All three files landed cleanly: `npm.nix` (new), `home.nix` (import added), `fish.nix` (function added).

**Open:** The 1Password secret reference path may need adjusting to match actual vault structure. Optional direnv auto-loading for workspace directories was documented in the plan but not implemented (manual user step). Changes are uncommitted.
