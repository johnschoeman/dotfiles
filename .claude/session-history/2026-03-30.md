## Build the add-project-skill catalog system

**Thread:** Wanted a way to scaffold project-local skill overrides from reusable templates instead of writing them from scratch each time. The core problem: global skills are generic, but each project needs slightly different conventions (commit formats, task thresholds, session logging behavior).

Designed and built `/add-project-skill` — a global skill with a file-based catalog at `claude/skills/add-project-skill/catalog/`. Each catalog entry is a directory with a `README.md` (description, detection hints, customization questions) and `.template` files with `{{placeholder}}` markers. The skill detects project context (stack, CLAUDE.md, existing skills), presents the catalog with recommendations, asks customization questions, and scaffolds adapted skills to `.claude/skills/`.

Built out 15 catalog templates: commit, task-board, task-groom, task-plan, task-update, weekly-planning, weekly-review, wrap-day, wrap-session, update-knowledge, update-session-log, test-me-on, today, content-capture, content-synthesize. Added a README.md with a summary table of all templates and their per-project customization points.

Tested the skill live on the dotfiles repo itself — scaffolded 8 project-local skills (commit, task-board, task-groom, task-plan, task-update, update-knowledge, update-session-log, wrap-session) adapted to the repo's conventions (suggest-only commits, priority-based tasks, 60-day stale threshold).

Key design choices: file-based discovery (no registry), `<!-- catalog: name v1 -->` version comments for audit tracking, templates as guidance not rigid substitution, `--audit` mode for checking existing skills against catalog updates.

Also confirmed no Evolv/synapse references exist in the dotfiles repo — the synapse project is separate in `~/workspace/evolv-synapse`.
