# Global Rules & Preferences

## Interaction Style
- Be extremely concise. Avoid conversational filler, apologies, or polite pleasantries (no "great question").
- Present plans clearly. Use actionable checklists.
- If a request is unclear, ask for clarification.

## Workflow & Safety
- **Never** auto-commit changes.
- Always run tests before announcing a task is complete.
- When creating new files, ask before creating directories.

## Self-Improvement
- After every coding session, if I correct a pattern, add it to this file to improve your instructions.
